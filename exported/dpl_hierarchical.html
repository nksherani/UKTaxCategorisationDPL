DPL Tag Classifier — Hierarchical 2-Step Approach (CPU)



Instead of classifying directly into 76 DPL tags (hard), we do it in two steps:

`
Description
│
▼ Level 1 — 9-class group classifier
Group (e.g. "Finance & Treasury")
│
▼ Level 2 — group-specific classifier (~5–13 classes)
DPL Tag (e.g. "DPL035")
`

Why this works better:
- Level 1 narrows the search space from 76 → ~5–13 candidates
- Level 2 classifiers only compete against similar tags → less confusion
- Confusable pairs like DPL006 vs DPL075 (both interest income) are in the same group, trained to discriminate each other

All models use TF-IDF + LinearSVC — trains in < 30 seconds total on CPU.

Structure


1. Load dataset
2. Define 9-group hierarchy
3. Train Level 1 — group classifier
4. Train Level 2 — one classifier per group
5. Build HierarchicalDPLClassifier pipeline
6. Evaluate — hierarchical vs flat comparison
7. Confusion analysis
8. Save all models
9. Inference helper

from sklearn.pipeline import Pipeline, FeatureUnion
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.svm import LinearSVC
from sklearn.preprocessing import LabelEncoder
from sklearn.metrics import classification_report, accuracy_score, f1_score

def build_tfidf_svc(C=1.0):
    """Reusable TF-IDF + LinearSVC pipeline."""
    return Pipeline([
        ("features", FeatureUnion([
            ("word", TfidfVectorizer(
                analyzer="word", ngram_range=(1, 3),
                min_df=2, max_features=80_000, sublinear_tf=True,
            )),
            ("char", TfidfVectorizer(
                analyzer="char_wb", ngram_range=(3, 5),
                min_df=3, max_features=40_000, sublinear_tf=True,
            )),
        ])),
        ("clf", LinearSVC(C=C, max_iter=2000, dual="auto")),
    ])

# One LabelEncoder + SVC pipeline per group
l2_clfs   = {}   # group_name -> fitted pipeline
l2_les    = {}   # group_name -> LabelEncoder for DPL tags in that group

t0 = time.time()
for group, tags in HIERARCHY.items():
    mask = trainval_df["group"] == group
    grp_df = trainval_df[mask]

    if grp_df.empty or grp_df["dpl_tag"].nunique() < 2:
        print(f"  SKIP {group} — not enough classes")
        continue

    le_tag = LabelEncoder()
    le_tag.fit(sorted(grp_df["dpl_tag"].unique()))

    X_grp = grp_df["description"].tolist()
    y_grp = le_tag.transform(grp_df["dpl_tag"])

    clf = build_tfidf_svc(C=2.0)   # slightly higher C for fine-grained discrimination
    clf.fit(X_grp, y_grp)

    l2_clfs[group] = clf
    l2_les[group]  = le_tag
    print(f"  ✓ {group:<40} {len(le_tag.classes_):>2} tags  "
          f"{len(grp_df):>5} samples")

print(f"\nLevel 2 total training time: {time.time()-t0:.1f}s")

5. Build HierarchicalDPLClassifier Pipeline

In [7]:
class HierarchicalDPLClassifier:
    """
    Two-stage classifier:
      Stage 1 — predict the 9 accounting groups
      Stage 2 — predict the DPL tag within that group

    Parameters
    ----------
    l1_clf    : fitted Level-1 group classifier
    le_group  : LabelEncoder for groups
    l2_clfs   : dict {group_name: fitted Level-2 classifier}
    l2_les    : dict {group_name: LabelEncoder for DPL tags}
    """

    def __init__(self, l1_clf, le_group, l2_clfs, l2_les):
        self.l1_clf   = l1_clf
        self.le_group = le_group
        self.l2_clfs  = l2_clfs
        self.l2_les   = l2_les

    def predict(self, descriptions: list[str]) -> list[str]:
        """Return predicted DPL tag for each description."""
        # Stage 1 — predict group for all at once
        group_ids  = self.l1_clf.predict(descriptions)
        group_names = self.le_group.inverse_transform(group_ids)

        results = [None] * len(descriptions)

        # Stage 2 — batch by group to keep it fast
        for group in set(group_names):
            indices = [i for i, g in enumerate(group_names) if g == group]
            batch   = [descriptions[i] for i in indices]

            if group not in self.l2_clfs:
                # Fallback: use most common tag in group
                fallback = self.l2_les[list(self.l2_les.keys())[0]].classes_[0]
                for i in indices:
                    results[i] = fallback
                continue

            tag_ids   = self.l2_clfs[group].predict(batch)
            tag_names = self.l2_les[group].inverse_transform(tag_ids)
            for i, tag in zip(indices, tag_names):
                results[i] = tag

        return results

    def predict_with_groups(self, descriptions: list[str]):
        """Return (predicted_group, predicted_dpl_tag) for each description."""
        group_ids   = self.l1_clf.predict(descriptions)
        group_names = self.le_group.inverse_transform(group_ids)
        dpl_tags    = self.predict(descriptions)
        return list(zip(group_names, dpl_tags))

for tag_a, tag_b, reason in confusion_pairs:

    if mask_ab.sum() == 0:
        continue

    # Confusion = predicted B when true is A, or predicted A when true is B

    print(f"  {tag_a} ↔ {tag_b}  {reason:<28} {flat_conf:>11} {hier_conf:>11}  {arrow}")

def load_hierarchical_classifier(save_dir="models/hierarchical"):
    """Re-load the full hierarchical classifier from disk."""
    with open(f"{save_dir}/hierarchy.json") as f:
        hierarchy = json.load(f)

    _l1_clf   = joblib.load(f"{save_dir}/l1_group_classifier.joblib")
    _le_group = joblib.load(f"{save_dir}/l1_group_label_encoder.joblib")

    _l2_clfs, _l2_les = {}, {}
    for group in hierarchy:
        safe = group.replace(" ", "_").replace("&", "and").replace(",", "")
        clf_path = f"{save_dir}/l2/{safe}_clf.joblib"
        le_path  = f"{save_dir}/l2/{safe}_le.joblib"
        if os.path.exists(clf_path):
            _l2_clfs[group] = joblib.load(clf_path)
            _l2_les[group]  = joblib.load(le_path)

    return HierarchicalDPLClassifier(_l1_clf, _le_group, _l2_clfs, _l2_les)

Summary



How the 2-step approach helps



| Issue | Flat classifier | Hierarchical classifier |
|---|---|---|
| DPL006 vs DPL075 (interest income) | Competes with all 76 tags | Only competes within Finance & Treasury (13 tags) |
| DPL003 vs DPL038 (audit vs legal) | Competes with all 76 tags | Only competes within Professional Services (5 tags) |
| DPL045 vs DPL074 (generic) | Easily confused | Separated: DPL045→Revenue, DPL074→Miscellaneous |

Error modes



The hierarchical approach can fail in two ways:
1. Level 1 error — wrong group predicted → guaranteed wrong DPL tag
2. Level 2 error — correct group, wrong tag within group

If Level 1 accuracy is very high (>97%), the hierarchical approach almost always wins.
If Level 1 accuracy is low, the flat classifier may outperform.
